# Transcript Step

<p style={{ color: '#a3a3a3', fontSize: '0.875rem', lineHeight: 1.8, marginBottom: '2rem' }}>
  One line in an agent's work log. The system thinks, acts, and reports — each step
  is visible, collapsible, and honest. Transparency builds trust.
</p>

---

## Why this matters

Faultline is autonomous — agents investigate issues, run commands, and apply fixes
without asking. But "automatic" should never feel like "invisible." Every action the
system takes is recorded as a transcript the user can inspect at any time.

This is Principle 2 in action: **Show the Machine Thinking.**

## Collapsible by default

Most steps are collapsed — the transcript is scannable at a glance. The user
expands individual steps when they're curious about the details. This keeps the
interface calm while maintaining full transparency.

<div style={{ background: '#141414', borderRadius: 12, padding: 16, margin: '1.5rem 0' }}>

    kind="tool-call"
    title="kubectl top nodes"

Click the chevron to expand. The output appears in a monospace block — machine
data in the machine voice.

    </div>
  ))}
</div>

A running step is the only one that demands attention. Everything else settles
into the background — completed work doesn't compete for focus.

---

## The rail

Steps are connected by a thin vertical rail — the same pattern as the
timeline, but thinner and more subdued. The transcript is a subordinate
view, not the primary narrative. The rail creates continuity without
pulling focus from the issue timeline above it.
