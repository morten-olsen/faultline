## Composable structure

Cards use sub-components for consistent internal layout:

- **CardHeader** — title + tags + trailing content (timestamps, actions)
- **CardBody** — the description
- **CardCallout** — inset highlight for context the user needs
- **CardFooter** — stage pill + navigation

---

## A needs-you card

The amber treatment is calm — it's a hand-raise, not a fire alarm.
"Needs you" means the pipeline is paused, waiting for a human to act.

<div style={{ margin: '1.5rem 0' }}>
  <Card variant="needs-you">
    <div style={{ display: 'flex', alignItems: 'center', gap: 8, marginBottom: 12 }}>
      <Hand size={15} style={{ color: '#fbbf24', opacity: 0.8 }} />
      <span style={{ fontSize: '0.875rem', fontWeight: 500 }}>NAS disk showing SMART warnings</span>
    </div>
    <CardBody>
      Disk 3 is reporting reallocated sectors. The array is degraded but online.
      This needs a physical disk swap.
    </CardBody>
    <div style={{ display: 'flex', gap: 8 }}>
      <Button variant="primary" size="sm">I'll replace it</Button>
      <Button variant="ghost" size="sm" iconRight={ArrowRight}>Details</Button>
    </div>
  </Card>
</div>

## A calm issue card

Most cards are just the system showing its work. No action needed.

<div style={{ margin: '1.5rem 0' }}>
  <Card variant="issue" interactive>
    <CardHeader trailing={<span className="text-xs font-mono text-text-muted">3m</span>}>
      <span style={{ fontSize: '0.875rem', fontWeight: 500 }}>Memory pressure on node-02</span>
      <Tag>kubernetes</Tag>
    </CardHeader>
    <CardBody>
      Above 80% memory. Low-priority workloads moved to node-01.
    </CardBody>
    <CardFooter>
      <StagePill stage="monitoring" />
      <Button variant="ghost" size="sm" iconRight={ArrowRight}>Details</Button>
    </CardFooter>
  </Card>
</div>

The visual weight is noticeably lower. Same structure, same components,
but the absence of the amber tint says everything: this one's handled.
