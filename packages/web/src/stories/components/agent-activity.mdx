# Agent Activity

## The agent is your colleague

In Faultline, agents do the work — they investigate, plan, execute, and monitor.
The AgentActivity indicator is how the user sees that work happening in real time,
embedded directly in the issue timeline.

<div style={{ background: '#141414', borderRadius: 12, padding: '1.5rem', margin: '1.5rem 0', borderLeft: '3px solid #3b82f6' }}>
  <p style={{ margin: 0, color: '#a3a3a3', fontStyle: 'italic', fontSize: '0.9375rem', lineHeight: 1.7 }}>
    "Automatic by default, never a loss of control."
  </p>
</div>

The stop button is always there when the agent is active. Control is never more
than one tap away. But it's quiet — a small square, not a big red button. The
default assumption is that the agent knows what it's doing.

## Working vs waiting

**Working** means the agent is actively running commands and making progress.
The blue pulse says "I'm on it."

**Waiting** means the pipeline is blocked on the user. The amber pulse mirrors
the "Needs you" pattern from the rest of the system — same color, same meaning.
The agent hasn't stopped — it's just paused until you act.

---

## The expand affordance

The chevron on the right opens the full **agent transcript** — the complete
record of what the agent thought, what it ran, and what it found. This is
the bridge between the high-level timeline ("Rebalanced workloads") and the
low-level details ("ran `kubectl drain`, moved 3 pods, freed 1.1 GB").

Most users won't expand most transcripts. But when they do, every detail
is there. That's the trust contract: you don't *have* to look, but you
always *can*.

---

## Placement

The indicator sits inside timeline entries — directly below the entry title.
It's subordinate to the timeline narrative but always visible. When an agent
is actively working, the indicator's subtle ring and pulse draw the eye
without competing with the "Needs you" cards above it.
