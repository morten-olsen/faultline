## Page anatomy

The detail page follows the same narrow column as the home page (`max-w-lg`).
No layout shift when navigating — the user feels like they stepped one pace
closer to the same view.

### Event types and their visual treatment

<div style={{ display: 'flex', flexDirection: 'column', gap: '0.75rem', margin: '1.5rem 0' }}>
  {[
    { icon: <Search size={15} style={{ color: '#60a5fa' }} />, label: 'Detected', desc: 'System noticed an anomaly. Blue dot.' },
    { icon: <Lightbulb size={15} style={{ color: '#60a5fa' }} />, label: 'Analysis', desc: 'System reasoning about what to do. Blue dot.' },
    { icon: <Wrench size={15} style={{ color: '#60a5fa' }} />, label: 'Action', desc: 'System did something. May include a command block.' },
    { icon: <CircleCheck size={15} style={{ color: '#4ade80' }} />, label: 'Outcome', desc: 'Result of an action. Green for success, red for failure.' },
    { icon: <RotateCcw size={15} style={{ color: '#fbbf24' }} />, label: 'Regression', desc: 'Fix didn\'t hold. Amber — honest, not hidden.' },
    { icon: <Hand size={15} style={{ color: '#fbbf24' }} />, label: 'Needs you', desc: 'System is blocked. Amber hand icon.' },
    { icon: <UserCheck size={15} style={{ color: '#5a5a5a' }} />, label: 'User action', desc: 'What the user did. Dimmed — it\'s history.' },
    { icon: <CircleCheck size={15} style={{ color: '#4ade80' }} />, label: 'Resolved', desc: 'Issue closed. Green checkmark.' },
  ].map(({ icon, label, desc }) => (
    <div key={label} style={{ display: 'flex', alignItems: 'center', gap: 12 }}>
      {icon}
      <div>
        <span style={{ fontSize: '0.8125rem', fontWeight: 500 }}>{label}</span>
        <span style={{ color: '#5a5a5a', fontSize: '0.8125rem' }}> — {desc}</span>
      </div>
    </div>
  ))}
</div>

**The timeline is always fully expanded.** No "show more" toggle. The full history is visible.
Collapsing would violate *"History is always accessible, never buried."*

**Command blocks** appear in monospace (`font-mono bg-white/4 rounded-lg`) when the system ran
a specific command. This is how the system shows its work precisely — the actual kubectl or
CLI equivalent of what it did. Builds trust without burying the user in logs.

## Infrastructure chips

<div style={{ display: 'flex', gap: 8, margin: '1.5rem 0', overflow: 'auto' }}>
  <ResourceChip kind="node" name="node-02" health="degraded" detail="Memory 83%" />
  <ResourceChip kind="node" name="node-03" health="degraded" detail="Memory 81%" />
  <ResourceChip kind="node" name="node-01" health="healthy" detail="Memory 61%" />
</div>

**Why chips and not a table?** Horizontal scroll on mobile. Each chip is self-contained —
resource icon, monospace name, health status, optional metric. The user scans without having
to parse rows and columns.

**Monospace names** signal "this is a specific thing you can look up." The visual switch from
proportional to mono is a consistent pattern across the system.

## The resolved footer

Only appears when `stage === 'resolved'`. A quiet close:

<div style={{ textAlign: 'center', color: '#5a5a5a', fontSize: '0.75rem', padding: '1.5rem 0', borderTop: '1px solid rgba(255,255,255,0.05)', margin: '1.5rem 0' }}>
  Resolved 25 minutes ago · Took 25 minutes from detection to close
</div>

Whisper-weight. The resolved state is already communicated by the stage pill — this is
just the accounting. No celebration, no fanfare. The system handled it. That's the whole point.

## Navigation

The top bar on the detail page differs from the home:

- **Left:** Back arrow — returns to the home feed
- **Right:** Overflow menu (mute, copy link, open in cluster dashboard)

No title in the bar — the issue title *is* the page heading. No breadcrumbs.
The user came from one place and goes back to one place.
