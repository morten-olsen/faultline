## How you get here

The transcript isn't a page you navigate to — it's a page you **drill into**.
Every timeline entry that involves agent work has an expand affordance. Tap it,
and you see the full story.

## Stopping an agent

<div style={{ background: '#141414', borderRadius: 12, padding: '1.5rem', margin: '1.5rem 0', borderLeft: '3px solid #5a5a5a' }}>
  <p style={{ margin: 0, color: '#a3a3a3', fontStyle: 'italic', fontSize: '0.9375rem', lineHeight: 1.7 }}>
    "Automatic by default, never a loss of control."
  </p>
</div>

The stop button is always accessible when an agent is active — both in the
transcript view (top bar) and in the timeline (agent activity indicator).
Stopping is immediate and non-destructive. The issue stays open and can be
picked up later.

Two places to stop:
- **In the timeline** — the <Square size={10} style={{ display: 'inline', verticalAlign: 'middle', color: '#5a5a5a' }} /> button on the agent activity indicator
- **In the transcript** — the "Stop agent" link in the top navigation bar

## Relationship to the timeline

The timeline is the **high-level narrative**: "Detected memory pressure",
"Rebalanced workloads", "Waiting for approval." Each entry is a chapter
heading.

The transcript is the **detailed notes**: every `kubectl` command, every
piece of reasoning, every error. It's one level deeper — the curious user's
view.
